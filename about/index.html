Infinite Message Scrolling App

By Jasmin Rubinovitz

The Task

Build a prototype that shows a potentially very long list of messages. Users should be able to swipe a message horizontally offscreen to dismiss it. Messages should load automatically so that a user scrolling slowly should never see the bottom of the list. The prototype should work well even after many messages have been loaded.

Implementation

Given a mock-up, my first goal was to implement a prototype with a User Experince that is identical to the one described, with a few added features I thought would be nice to have on an infinite scrolling interface. For the prototype implementation I chose to use React.js, with the Google Material UI guidelines, implemented with the react-material-ui framework. I use react-swipeable to handle the messages swiping events.

Features

Infinite Scroll

The App loads 20 messages at a time. As the user scrolls through the messages, when she reaches a specific heights, more messages are loaded through the API. This give an infinite scroll experience. Since scrolling can become very long, and the user might not be interested in scrolling back through all the messages she already saw. I added a "Scroll to top button" clicking it shows an animation and the app automatically scrolls to the top message.

Message Dismiss

As a message is swiped to the right, it becomes transparent, signaling to the user that it is going to be deleted. When the swipe is done, the message is animated out of the screen and disappears. A toast appears at the bottom, telling the user that the message was deleted and allowing her to undo the deletion and return the message.

Bookmark

Infinite scrolling can make it hard for the user to find items she liked. So I decided to add a bookmarking option. Using the swipe left gesture, the user can star and un-star messages.

Clicking on the star icon at the header bar filters the messages. It toggles a presentation of all messages or only the messages that were starred. Being able to easily filter a long list of items is important and can save time and effort for the user, making the experience easier and fun.
